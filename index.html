高效工作流，我是怎么做的

最近做了几个项目之后，我慢慢摸索出一套并行工作节奏——不是同时做多件事让自己焦虑， 而是让每个工具在各自的轨道上跑，我负责在中间做决策和连接。

多任务并行
一边生成，一边阅读

让 Claude 跑一个比较慢的任务时，切到 Anti-Gravity 做深度阅读， 或者处理另一件完全不同的事。不等待，不空转。

资源切换
Token 用完？无缝换一个

单个账号额度耗尽时，浏览器端切换登录 + 同步调整 IDE 接入方式， 任务不中断，继续往下走。

云端部署
RunCloud 降低门槛

借助 RunCloud 等管理面板，云服务器的部署和管理变得像操作本地一样直观， 不需要精通运维也能快速上线。

核心节奏
做 → 看结果 → 调整

Web coding 最大的好处是反馈极快。不需要等一个「完美计划」， 先跑起来，看到真实结果，再决定下一步。

大模型的思维方式是用最少的 token 完成任务——跟大脑「偷懒」的本质一样。 当你直接告诉它你想要什么，中间很多过程就会被合理简化。

我有 ADHD，有时候会不自觉地把任务想得很复杂。但当我开始用「直接成为」的思维方式， 就会直接问 Agent：「我能不能做这件事？需要哪些工具？你能帮我什么？」

这种思维不只是用 AI 的技巧，它改变了我跟人合作的方式—— 我会更清楚地去想：对方想要从我这里得到什么？我们最终想做成什么？中间的 gap 有多大？

02

AI 时代的团队协作模式

当团队里每个人都有自己的 Coding Agent，协作模式会发生一些结构性的变化。 我们在调研项目里用了「定量 + 定性结合」的方式，效能提升非常明显。

定量分析 → 每位成员负责不同方向，AI 大规模处理数据 定性分析 → 统一可视化 Dashboard，线下沟通，划线标注达成共识 产出 → 简洁的观点文档，支撑后续决策

最直观的感受：每位成员配备一个 Coding Agent，团队实际效能从 N 提升到 N×2。 不是 AI 替代了人，而是每个人都多了一个随时在线、不会疲惫的「执行搭档」。

「一人一机」不是噱头——它真实地改变了谁能做什么、做得多快、做得多好。

项目一：AI 市场机会调研平台

这个项目的起点是一个问题：哪些 AI 使用场景能让小白用户一句话完成生产级输出，并且愿意自发分享？ 我们用 Agent 扫了市场，抓取数据，然后搭了一个协作平台让团队共同分析。

Research · Flask · GitHub Pages
AI 市场机会调研平台

45 个 AI 产品病毒场景的调研、评分与协作分析平台。支持多人在线编辑、 乐观锁冲突检测、场景筛选排序、Excel 导出，以及无需安装环境的 GitHub Pages 只读版。

一步步是怎么做的？
  1. 先定问题：我们想找什么？用一句话写清楚调研目标，让 Agent 帮我们分解成可执行的搜索策略。
  2. 信号扫描：让 Agent 搜 Product Hunt、小红书、抖音等平台的热门 AI 产品，抓取近期在爆的案例。
  3. 结构化评分：把每个场景按「小白友好度 / 病毒传播力 / 技术匹配度 / 市场空白 / 变现潜力」五个维度打分，存入 Excel。
  4. 搭可视化平台：用 Flask 起一个本地服务，让团队成员都能在浏览器里看到场景卡片、编辑评分，不需要碰代码。
  5. 多人协作：同时支持本地局域网实时协作 + GitHub Pages 静态版，保证每个人都能参与，无论有没有开发环境。

项目二：Marketing Design Agent

这个项目是 Web Coding 方式最直接的体现——我想把一套设计工作流「落地」成一个真正能用的工具， 而不只是一堆文档或者 prompt。

核心思路是把设计任务分成四个复杂度级别，然后为每个级别配置对应的 skill 和工作流。 这样不管用户提什么需求，Agent 都能自动路由到合适的处理逻辑。

用户输入 → 意图识别 → 任务分级 Level 1 简单图片 → 直接生图，无文案需求 Level 2 主图 / 场景图 → 可选文案（卖点标签） Level 3 商详海报 → 必须文案（每图配 narrative） Level 4 创意海报 → 必须文案（完整营销文案）
Design Agent · Next.js · Skills System
Marketing Design Agent

一个结构化的设计 + 文案生成 Agent，内置 DNA 生成器（提取品牌调性）、 任务路由器、Identity 一致性保持机制，以及按任务类型分级的 skill 库。

Next.js TypeScript Skill System DNA Generator Image API
查看项目详情
一步步是怎么做的？
  1. 先想清楚「设计任务」有哪些类型——从最简单的产品图到最复杂的创意海报，用复杂度分级来管理不同的处理逻辑。
  2. 搭 DNA 生成器：让 Agent 分析竞品样本或品牌素材，提取出标准化的「文案 DNA」和「设计 DNA」，作为后续生成的基础。
  3. 写任务路由器：一个统一的分发逻辑，根据用户触发词自动判断该走哪条流程，调哪个 skill。
  4. 搭 Identity 保持机制：产品外形、Logo、品牌色——不让 AI 随意改，严格控制一致性（产品图用图生图 / 抠图，Logo 后期叠加）。
  5. 用 Web Coding 方式部署：写好 skill 文档，配 Next.js 前端，直接在本地跑起来看效果，快速迭代。

项目三：图像分层与修复工具

这个是我觉得最「好玩」的一个 demo——把一张合成好的设计稿拆回「素材层 + 背景层 + 文字层」， 或者反过来，把某一层修复掉。本质上是把阿里云的图像 API 和 Gemini 的 inpainting 能力拼在一起。

Image Processing · Streamlit · Aliyun + Gemini
Image Layer Separation & Inpainting

两种模式：Layer Editor（分层编辑，提取主体 / 背景 / 文字各层） 和 UI Deconstructor（把 UI 截图拆解成可编辑的组件层）。 本地 Streamlit 界面，直接上传图片就能用。

一步步是怎么做的？
  1. 先选工具：阿里云有抠图 API（主体识别），302.ai 有 Gemini 的图像修复能力——把它们拼在一起用。
  2. 搭 Streamlit 界面：不需要前端开发，Python 写 UI，上传图片 → 选择操作 → 看结果，整个交互几十行代码搞定。
  3. Layer Editor 模式：上传一张图，调用阿里云 API 抠出主体，分离背景层，再用 Gemini 对某一层做 inpainting 修复。
  4. UI Deconstructor 模式：把一张 UI 截图分析成组件结构，让设计师能重新编辑各个层级。
  5. 本地跑，立刻看效果——这就是 Web Coding 的最大好处：不用部署，不用等，做完就能用。

都是「有个想法，直接跟 Agent 说，跑起来看看」。
