核心思路

把「合成好的图」拆回来

做设计时常有一个痛点：拿到一张合成好的图，想单独修改某一层—— 比如换背景、替换产品、去掉文字——但找不到原始分层文件。

这个工具做的事就是：把已经合成的图拆回各个层， 或者对某一层做 AI 修复，再重新合成。

核心洞察：阿里云有成熟的主体识别 + 抠图 API， Gemini 有 inpainting 能力——把它们拼在一起， 就能完成「分层 → 修改 → 重合成」的完整流程， 而且不需要 Photoshop。

使用模式

两种操作方式

Mode 1
Layer Editor

上传一张图，自动分离主体层、背景层、文字层。 可以单独对某一层做修复或替换，再重新合成输出。

Mode 2
UI Deconstructor

把一张 UI 截图分析成组件结构—— 哪里是导航、哪里是卡片、哪里是按钮—— 让设计师能重新编辑各个层级。

技术架构

两个 API 的分工

阿里云 API 主体识别 → 精准抠图 → 透明背景 PNG 分割置信度 + 边缘羽化处理 Gemini API inpainting → 对指定区域做 AI 修复 (via 302.ai) 理解图像内容 → 上下文一致性填充 Python + PIL 图层合并 / 蒙版处理 / 透明度混合 NumPy 矩阵操作 → 精细像素级控制 Streamlit 本地 UI → 上传 / 预览 / 下载 几十行代码完成完整交互界面
完整过程

一步步是怎么做的

  1. 先选工具：阿里云有抠图 API（主体识别），302.ai 有 Gemini 的图像修复能力——把它们拼在一起用。
  2. 搭 Streamlit 界面：不需要前端开发，Python 写 UI，上传图片 → 选择操作 → 看结果，整个交互几十行代码搞定。
  3. Layer Editor 模式：上传一张图，调用阿里云 API 抠出主体，分离背景层，再用 Gemini 对某一层做 inpainting 修复。
  4. UI Deconstructor 模式：把一张 UI 截图分析成组件结构，让设计师能重新编辑各个层级。
  5. 本地跑，立刻看效果——这就是 Web Coding 的最大好处：不用部署，不用等，做完就能用。
这个项目的意义

拼接能力，而不是造能力

这个 demo 最有意思的地方不是技术有多复杂，而是它展示了一种思路： 你不需要从零训练一个模型，也不需要自己实现抠图算法。

把现有的成熟 API 组合起来，用 Python 做胶水， 用 Streamlit 快速出界面——几天时间就能做出一个真正可用的工具。

这是 Web Coding 最直接的体现：有想法，直接组装，立刻能用。

最好的工具往往不是从零造出来的，
而是把对的东西拼在了一起。
