核心思路

把设计工作流变成一个工具

这个项目的起点不是「我要做一个 AI 设计工具」，而是「我想把自己用 AI 做设计的那套流程落地成能给别人用的东西」。

最核心的挑战是：不同的设计任务复杂度差很多。做一张简单产品图和做一套品牌创意海报，完全是两件事。 所以我用复杂度分级来管理不同的处理逻辑。

关键决策：不让 AI 随意发挥品牌元素。产品外形、Logo、品牌色—— 严格通过图生图、抠图 + 后期叠加来控制一致性，而不是依赖 prompt 描述。

任务路由

四个复杂度级别，自动路由

用户不需要知道背后有多少条流程——只需要描述想要什么， Agent 根据触发词自动判断该走哪条路径。

Level 1
简单图片

直接生图，无文案需求。适合背景图、素材图、氛围图。

Level 2
主图 / 场景图

可选文案（卖点标签）。需要产品 Identity 保持，用图生图处理。

Level 3
商详海报

必须文案（每图配 narrative）。从 DNA 库中调取对应的文案风格。

Level 4
创意海报

必须文案（完整营销文案）。调用完整的创意流程，包含概念生成、视觉方向、文案。

品牌 DNA 系统

提取一次，复用无数次

每个品牌第一次使用时，Agent 会分析竞品样本或上传的品牌素材， 提取「文案 DNA」和「设计 DNA」，存入 Library。

文案 DNA 品牌语气 / 句式偏好 / 关键词库 / 禁用词 目标受众画像 / 核心卖点优先级 设计 DNA 主色 / 辅色 / 字体规范 构图偏好 / 留白风格 / 图片氛围 使用方式 新任务自动引用 → 保持视觉一致性 可手动编辑更新 → 品牌升级时同步

这样的好处是：第二次、第三次做同一个品牌的物料时， 不需要重新解释品牌调性，Agent 已经「认识」这个品牌了。

完整过程

一步步是怎么做的

  1. 先想清楚「设计任务」有哪些类型——从最简单的产品图到最复杂的创意海报，用复杂度分级来管理不同的处理逻辑。
  2. 搭 DNA 生成器：让 Agent 分析竞品样本或品牌素材，提取标准化的「文案 DNA」和「设计 DNA」，作为后续生成的基础。
  3. 写任务路由器：一个统一的分发逻辑，根据用户触发词自动判断该走哪条流程，调哪个 skill。
  4. 搭 Identity 保持机制：产品外形、Logo、品牌色——不让 AI 随意改，严格控制一致性（产品图用图生图/抠图，Logo 后期叠加）。
  5. 用 Web Coding 方式部署：写好 skill 文档，配 Next.js 前端，直接在本地跑起来看效果，快速迭代。

工具的价值不在于它有多智能，
而在于它是否真的符合工作流——
让设计这件事门槛更低，而不是更复杂。
